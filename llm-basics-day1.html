① Transformer 架构 / Transformer Architecture

[LLM][架构][基础]

// 核心概念

Transformer 是 2017 年 Vaswani 等人提出的、完全基于 self-attention 的序列建模架构，彻底取代了 RNN/CNN 在 NLP 的主导地位。本质突破：将"信息的远距离传递"从 RNN 的 $O(n)$ 串行路径压缩到 $O(1)$ 的并行可达——任意两个 token 一步互通，这是现代 LLM 智能涌现的物理基础，也是 GPU 大规模训练成为可能的工程基础。

// 原理详解

一个标准 Transformer Block 由两个子层组成，每个子层都包裹在残差连接 + 归一化中：

  • Multi-Head Self-Attention：负责 token 间的关系建模（句法、共指、远距离依赖）
  • Position-wise FFN：两层 MLP（隐藏维度通常 = 4×d_model），完成非线性变换，并承担"知识存储"的角色（Geva et al. 证明 FFN 是 key-value memory）
$\text{Block}(x) = x + \text{FFN}(\text{LN}(x + \text{Attn}(\text{LN}(x))))$

现代 LLM (GPT/Llama/Claude) 的标配演进：

  • Decoder-only：抛弃原 encoder-decoder，仅保留 causal mask 的 decoder，简化训练 + 适配自回归生成
  • Pre-LN：LayerNorm 前置于残差路径内部，训练稳定性远优于 Post-LN（避免深层梯度爆炸）
  • RMSNorm：去掉 LayerNorm 中的均值减法和 bias，计算量降 30%，效果持平
  • SwiGLU 激活：$\text{SwiGLU}(x) = \text{Swish}(xW_1) \odot (xW_2)$，比 ReLU/GELU 提升 1-2 个 perplexity 点

规模参考（Llama-3-70B）：80 层、d_model = 8192、64 个 attention head（其中 8 个 KV head, GQA）、FFN 中间维度 28672。总参数 70B，其中约 70% 在 FFN，30% 在 attention 投影。

架构示意：

Input Tokens (B, S)
   ↓
[Token Embedding]  →  (B, S, d_model)
   ↓
┌─ Transformer Block × N ───────────┐
│   RMSNorm                          │
│   Multi-Head Attention (causal)    │ ─┐
│   RoPE on Q,K                      │  │ residual
│   ─────────────────────────        │ ←┘
│   RMSNorm                          │
│   FFN (SwiGLU, 4× expansion)       │ ─┐
│   ─────────────────────────        │ ←┘ residual
└────────────────────────────────────┘
   ↓
[Final RMSNorm → LM Head (tied embedding) → Softmax]
   ↓
Next Token Distribution P(x_{t+1} | x_{≤t})

// 代码示例

import torch, torch.nn as nn, torch.nn.functional as F

class RMSNorm(nn.Module):
    def __init__(self, d, eps=1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(d))
        self.eps = eps
    def forward(self, x):
        return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps) * self.weight

class SwiGLU(nn.Module):
    def __init__(self, d, ff):
        super().__init__()
        self.w1, self.w2, self.w3 = nn.Linear(d,ff,bias=False), nn.Linear(d,ff,bias=False), nn.Linear(ff,d,bias=False)
    def forward(self, x):
        return self.w3(F.silu(self.w1(x)) * self.w2(x))   # silu = swish

class Block(nn.Module):
    def __init__(self, d=4096, h=32, ff=11008):
        super().__init__()
        self.norm1, self.norm2 = RMSNorm(d), RMSNorm(d)
        self.attn = nn.MultiheadAttention(d, h, batch_first=True, bias=False)
        self.ffn = SwiGLU(d, ff)
    def forward(self, x, mask):
        h = self.norm1(x); x = x + self.attn(h,h,h,attn_mask=mask,need_weights=False)[0]
        return x + self.ffn(self.norm2(x))

  • Touvron et al., 2023. LLaMA: Open and Efficient Foundation Language Models — 现代 decoder-only 工程范本

// 跨学科联想

Transformer 的并行可达性，在数学结构上对应大脑皮层的全局工作空间理论 (Global Workspace Theory, Baars-Dehaene)：意识的产生需要将局部信息"广播"到全脑，attention 即此广播机制的离散化表达。从复杂性科学看，Transformer 是一个稠密耦合的动力系统，每个 token 的状态由所有其他 token 共同决定，与 Kuramoto 耦合振子的同步现象同构。从佛学角度，"诸法因缘生"——每个 token 的语义由全体上下文共同因缘构成，无独立自性，正是 attention 机制的隐喻。

// 实践场景

经典应用：所有现代 LLM (GPT-4、Claude、Gemini、Llama) 的核心引擎；多模态模型 (CLIP/Sora) 的统一架构。
超级个体场景：深入理解 Transformer 让你能精准判断模型能力边界——例如知道长序列中间位置 attention 分配稀疏（"lost in the middle"现象），就能在设计 RAG/Agent 工作流时把最关键信息放在 context 的头尾而非中间，显著提升召回质量。

EN Summary: The Transformer replaces sequential recurrence with parallel self-attention, achieving $O(1)$ path length between any two tokens. Modern decoder-only LLMs (GPT/Llama/Claude) layer Pre-LN/RMSNorm, SwiGLU FFN, and rotary positions atop the original design. FFN stores knowledge; attention routes it. This dense coupling is what enables scaling laws and emergent capabilities.
💭 思考题：如果 Transformer 的本质是"全连接信息广播"，那么人脑用稀疏的局部连接 + 远距离白质纤维束实现了类似功能，能耗却只有 20W。"稀疏 vs 稠密"的差异是否暗示了下一代 LLM 架构的方向（如 MoE、Mamba）？

② Self-Attention 机制 / Self-Attention Mechanism

[LLM][核心][数学]

// 核心概念

Attention 是一次"软查表 (differentiable lookup)"：用 query 向量去匹配所有 key 向量，按相似度对 value 加权求和。它让模型在每一层都能动态决定"看哪里"。关键地位：在 Transformer 中，attention 是唯一支持 token 间交互的算子（FFN 是 token-wise 独立的），因此所有"上下文理解"能力都来自它。

// 原理详解

$\text{Attention}(Q,K,V) = \text{softmax}\!\left(\dfrac{QK^\top}{\sqrt{d_k}} + M\right) V$

三个关键设计：

  • 缩放因子 $\sqrt{d_k}$：内积方差随维度增长，未缩放会使 softmax 饱和、梯度消失。$\sqrt{d_k}$ 让方差保持 $O(1)$
  • Causal Mask $M$：decoder-only LLM 中，将上三角设为 $-\infty$，保证 token $t$ 只看 $\le t$ 的位置（自回归性质）
  • Multi-Head：将 d_model 拆成 h 份并行做 attention，最后 concat。不同 head 自发学到不同关系——句法 head、共指 head、远距离 head 等（Anthropic 的 induction head 研究即基于此）

现代生产级变种：

  • GQA (Grouped-Query Attention)：多个 Q head 共享一组 K/V head，KV cache 显存降 4-8 倍。Llama-3 / Mistral / Claude 全员采用
  • MQA (Multi-Query)：所有 Q 共享 1 组 KV，极端压缩，质量略损
  • FlashAttention v2/v3：IO-aware tiling，将 $O(n^2)$ 显存复杂度降到 $O(n)$，速度提升 2-4×，现已成为 PyTorch 2.0+ 默认实现
  • Sliding Window / Sink Attention：长上下文模型 (Mistral, Gemini) 用局部+少量全局 token 进一步降本

复杂度对比：原始 attention 显存 $O(n^2)$、计算 $O(n^2 d)$；FlashAttention 显存 $O(n)$、计算不变但常数小数倍。

// 代码示例

import torch, torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, causal=True):
    """Q,K,V: (B, H, S, d_k) — 教学版"""
    d_k = Q.size(-1)
    scores = (Q @ K.transpose(-2, -1)) / d_k**0.5         # (B,H,S,S)
    if causal:
        S = scores.size(-1)
        mask = torch.triu(torch.ones(S,S,device=Q.device,dtype=torch.bool), diagonal=1)
        scores = scores.masked_fill(mask, float('-inf'))
    attn = F.softmax(scores, dim=-1)
    return attn @ V                                        # (B,H,S,d_k)

# 生产：PyTorch 2.0+ 一行调用，自动调度 FlashAttention-v2 内核
out = F.scaled_dot_product_attention(Q, K, V, is_causal=True)

  • Dao et al., 2022. FlashAttention: Fast and Memory-Efficient Exact Attention
  • Ainslie et al., 2023. GQA: Training Generalized Multi-Query Transformer Models

// 跨学科联想

Attention 的 softmax 权重分配与神经科学中的"选择性注意"同构：丘脑-皮层回路通过抑制无关信号、放大相关信号实现注意聚焦（Posner 注意网络）。从量子力学看，$Q K^\top$ 内积像"波函数内积"——衡量两态重叠；softmax 则像 Born 规则将振幅化为概率。从佛学看，attention 即"作意"(manaskāra)——心识聚焦的最基础操作，五遍行心所之一。

// 实践场景

经典应用：翻译/摘要中的远距离依赖、代码补全中的变量追踪、对话中的指代消解。
投资场景：构建多 Agent 投资分析系统时，可借鉴 attention 机制——让"决策 Agent"在多个"分析师 Agent"输出之间学习权重（而非简单平均或投票），对高置信信号自动放大、低质量噪声自动抑制，类似 Ray Dalio 的 "believability-weighted decisions"。

EN Summary: Self-attention computes $\text{softmax}(QK^\top/\sqrt{d_k})V$, a differentiable soft-lookup. Multi-head splits the operation into parallel subspaces; each head learns a different relation type. Production systems use GQA to shrink KV cache and FlashAttention to make memory linear in sequence length. Attention is the only inter-token operator in a Transformer — every contextual capability is mediated by it.
💭 思考题：Softmax 在低 temperature 下倾向"赢者通吃"（分布很尖），这是 attention 的优点（聚焦）还是缺陷（错过弱信号）？人类灵感跳跃常源于关注"低概率关联"——下一代 attention 是否需要某种"创造性温度"机制？

③ Tokenization 分词 / Tokenization (BPE / SentencePiece)

[LLM][预处理][工程]

// 核心概念

Tokenization 把原始字符串切分成模型词表中的离散单元 (token)，是 LLM 的"感知器官"——决定模型能看到什么、看不到什么。重要性常被低估：分词边界直接决定模型的算术能力、跨语言公平性、多模态融合粒度、甚至 prompt injection 攻击面。

// 原理详解

主流方案是 BPE (Byte-Pair Encoding) 及其变种 BBPE (Byte-level BPE，GPT/Llama 采用)：

  1. 初始词表 = 256 个字节（保证无 OOV）
  2. 在大语料上统计相邻 token 对的频率，将最频繁的对合并为新 token，写入合并规则表
  3. 重复至目标词表大小（GPT-4 约 10 万，Llama-3 约 12.8 万，Gemini 约 25.6 万）

关键性质 & 实战陷阱：

  • 无 OOV：字节级 fallback 保证任意 unicode 可编码
  • 语言不公平：英文 1 token ≈ 4 字符；中文 1 字 ≈ 1.5-2 token；缅甸语/泰米尔语可达 1 字 ≈ 6+ token——直接影响 API 成本与延迟
  • 数字陷阱："12345" 可能切成 "12" + "345"，算术不稳。Llama-3 / GPT-4 已专门把数字按单字符切分
  • "strawberry" 中有几个 r：词级 token 让 LLM 无法"看到"内部字符——这是经典认知盲区，根源就在 BPE
  • SentencePiece (Unigram)：Llama/T5 使用的端到端学习版 BPE，无需预切词，对多语言更友好

// 代码示例

# pip install tiktoken
from tiktoken import get_encoding

enc = get_encoding("cl100k_base")  # GPT-4 tokenizer
text = "Transformer changed everything. 注意力即一切。"
ids = enc.encode(text)
print(f"chars={len(text)}, tokens={len(ids)}")
# chars=27 (含中文), tokens=14

for i in ids:
    print(f"  {i:>6}  → {enc.decode([i])!r}")
# 'Transform' + 'er' + ' changed' + ' everything' + '.'
# + ' 注' + '意' + '力' + '即' + '一' + '切' + '。'
# 观察：英文复合词被切碎，中文常一字一 token，效率显著差于英文

// 关键论文

  • Sennrich et al., 2016. Neural Machine Translation of Rare Words with Subword Units — BPE 引入 NLP
  • Kudo & Richardson, 2018. SentencePiece: A simple and language independent subword tokenizer

// 跨学科联想

Tokenization 类似认知科学中的"知觉单元化" (perceptual chunking)——大脑把连续输入切成音节、词、短语等有意义的块。Miller "7±2" 工作记忆容量受 chunk 大小调制：chunk 越好，承载越多。从复杂性科学看，BPE 合并规则是自组织临界 (SOC) 的体现：词表呈 Zipf 分布涌现，与自然语言的统计规律一致——这是数据驱动的"涌现结构"。

// 实践场景

经典应用：所有 LLM 输入输出的预/后处理；API 计费、context 长度计算。
育儿/教育场景：理解 tokenization 后，可与孩子用 OpenAI Tokenizer 网页演示——"为什么 ChatGPT 数不清 strawberry 里的 r？"。这是绝佳的"AI 局限性"启蒙素材：让孩子理解 AI 受具体机制约束、并非全知，培养批判性使用 AI 的能力，而非盲目崇拜或恐惧。

EN Summary: Tokenization splits raw text into subword units via BPE / SentencePiece. Byte-level BPE guarantees no OOV. Token granularity affects arithmetic, multilingual fairness, and API cost — Chinese consumes ~2× more tokens than English for equivalent content. The "strawberry r-count" failure is a tokenization artifact, not a reasoning bug.
💭 思考题：若未来 LLM 直接以"字节/像素"为单位（如 ByT5, Mamba-byte, Megabyte），会获得什么？牺牲什么？这是否让 AI 的"感知"更接近人类的连续知觉，还是更远离人类的"概念思维"？

④ 位置编码 RoPE / Rotary Positional Embedding

[LLM][架构][长上下文]

// 核心概念

Self-Attention 本身是置换不变 (permutation invariant) 的——打乱输入顺序不改变输出（除 mask 影响）。位置编码是注入"顺序信息"的唯一通道，决定模型如何理解时间、因果、远近。战略地位：它是 long-context 能力的核心瓶颈，几乎所有 100K+ context 模型的突破都来自位置编码的改进。

// 原理详解

三代演进：

  1. 正弦位置编码 (Sinusoidal, 2017)：原始 Transformer 用固定函数 $PE_{(pos,2i)}=\sin(pos/10000^{2i/d})$，加到 token embedding。问题：外推性差，超过训练长度性能骤降。
  2. Learned PE (GPT-2/BERT)：直接学一个 $(\text{max\_len} \times d)$ 矩阵。问题：完全无外推能力。
  3. RoPE (Rotary Position Embedding, 2021) ⭐ 现代主流：将 query/key 每对维度看作复平面上的二维向量，按位置 $m$ 旋转角度 $m\theta_i$，其中 $\theta_i = 10000^{-2i/d}$。
$\langle q_m, k_n \rangle_{\text{RoPE}} = \text{Re}\!\sum_i (q_m^{(i)} e^{im\theta_i})\overline{(k_n^{(i)} e^{in\theta_i})}$
→ 内积只依赖 $m-n$（相对位置！）

RoPE 的杀手锏：

  • 相对位置感知：QK 内积只依赖距离 $m-n$，符合语言的平移不变性
  • 无额外参数：纯函数变换，不增加可训练权重
  • 可外推：通过 Position Interpolation / NTK scaling / YaRN 等技术外推到训练长度的 4-32 倍——这是 Llama-3 从 8K 扩到 128K、Claude 达到 200K-1M context 的核心技术

RoPE 外推系列对比：

  • Position Interpolation (PI)：线性压缩位置 → 简单但牺牲高频信息
  • NTK-aware scaling：仅调整 base，保留高频 → 零样本可用
  • YaRN：分频段不同缩放 + 注意力温度修正 → 当前 SOTA

// 代码示例

import torch

def rope(x, base=10000):
    """x: (B, H, S, D), D 必须是偶数。返回旋转后的 x。"""
    B, H, S, D = x.shape
    half = D // 2
    # 频率：高维分量旋转慢，低维分量旋转快
    freq = 1.0 / (base ** (torch.arange(0, half, device=x.device).float() / half))
    pos = torch.arange(S, device=x.device).float()
    angles = pos[:, None] * freq[None, :]                # (S, D/2)
    cos, sin = angles.cos(), angles.sin()
    x1, x2 = x[..., :half], x[..., half:]
    # 二维旋转：[x1, x2] → [x1·cos − x2·sin, x1·sin + x2·cos]
    return torch.cat([x1 * cos - x2 * sin,
                      x1 * sin + x2 * cos], dim=-1)

# 应用：只对 Q 和 K 做 RoPE，对 V 不做（V 不参与位置匹配）
Q_rot, K_rot = rope(Q), rope(K)
# 然后正常 attention： softmax(Q_rot @ K_rot.T / sqrt(d)) @ V

// 关键论文

  • Su et al., 2021. RoFormer: Enhanced Transformer with Rotary Position Embedding
  • Peng et al., 2023. YaRN: Efficient Context Window Extension of Large Language Models

// 跨学科联想

RoPE 用"相位旋转"编码位置，与量子力学中的相位编码本质同构——量子态演化 $|\psi(t)\rangle = e^{-iHt/\hbar}|\psi(0)\rangle$、傅立叶变换、量子计算的相位干涉都使用同一数学结构 $e^{i\theta}$。从神经科学看，海马体的 theta phase coding（位置细胞放电相对于 theta 节律的相位编码空间位置/时间）几乎是 RoPE 的生物对应物。这种独立涌现的巧合提示："相位是编码序最自然的数学结构"——可能是宇宙级别的设计模式。

// 实践场景

经典应用：长文档问答、代码仓库级理解 (Cursor/Claude Code)、视频时序建模、DNA 序列分析。
超级个体场景：构建"个人知识库 Agent"时，理解 RoPE + YaRN 能让你判断给定模型在 long context 上的真实能力——某些模型号称 1M context，但在 100K+ 位置 retrieval 准确率骤降（NIAH/RULER 测试可见）。这决定了架构选择：长上下文一次性吞入 vs RAG 分块检索 vs 分层 summary。

EN Summary: Self-attention is permutation-invariant; positional encoding is the sole channel for order. RoPE encodes position as complex-plane rotation, so the QK inner product depends only on relative offset $m-n$. Combined with NTK / YaRN scaling, RoPE underpins the leap from 8K to 1M-token contexts in modern LLMs. The mathematical structure mirrors quantum phase encoding and hippocampal theta-phase coding.
💭 思考题：人脑无显式"位置编码"，却能处理长达数小时的故事——因为"上下文"已被压缩到 episodic memory（事件记忆），而非保留原始 token。下一代 AI 的"长上下文"，是该继续暴力扩 window，还是该学习压缩—检索—重构？这关乎你为孩子设计 AI 学习助手时——选择"塞入整本书"还是"分章记忆+检索"。
